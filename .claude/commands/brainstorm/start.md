# /brainstorm:start

Follow this workflow:
1. Read `AGENTS.md`, `README.md`, and `doc/specs/README.md` before proposing anything.
2. Ground ideas in the current repository, the hackathon theme, constraints, and likely demo value.
3. Create `doc/specs/<slug>/01-brainstorm.md` from the brainstorm template if it does not exist.
4. Write 3-5 concrete ideas with user value, feasibility, demo hook, and key unknowns.
5. Recommend starting with the `cheap` runtime profile unless deep technical synthesis is necessary.

If a required document is missing, create it from the matching template in `doc/specs/_templates/`.
