# /brainstorm:tasks

Follow this workflow:
1. Read `02-specification.md` and write `03-tasks.md` with executable tasks.
2. Treat `03-tasks.md` as the source of truth for progress and status.
3. Split work into teammate-safe items: ideation, research, handoff, implementation, validation.
4. Call out which tasks are suitable for cheap, deep, or local runtime profiles.

If a required document is missing, create it from the matching template in `doc/specs/_templates/`.
