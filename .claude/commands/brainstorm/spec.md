# /brainstorm:spec

Follow this workflow:
1. Read `01-brainstorm.md` and any existing repo context before writing a spec.
2. Create or update `doc/specs/<slug>/02-specification.md` using the specification template.
3. Resolve obvious ambiguities in favor of a shippable hackathon demo.
4. Keep acceptance criteria concrete and implementation-light enough for teammates to review.

If a required document is missing, create it from the matching template in `doc/specs/_templates/`.
